# Front end

<hr>

## 프론트엔드 (Front end)란

웹사이트나 앱에서 사용자가 직접 눈으로 보고 상호작용하는 영역(화면, UI/UX)을 개발하는 기술이다.

> - **사용자 접점** : 웹 브라우저를 통해 사용자가 마주하는 **화면 전체**를 뜻한다. <br>
> - **주요 역할** : 시각적 요소 구성, 사용하기 쉬운 인터페이스 (UI) 및 사용자 경험 (UX)을 구현한다.<br>
> - **기술 스택** : HTML(구조), CSS(스타일), JavaScript(동작) 등으로 구성된다.

## 프론트 엔드 활용 예시

> - **웹 페이지 레이아웃** : 웹 사이트 접속 시 보이는 **메뉴바, 배너,푸터**를 구성한다.<br>
> - **상호작용** : 버튼 클릭 시 동작, 폼 데이터 입력, 스크롤 반응, 팝업창 등이 있다.<br>
> - **데이터 시각화** : 그래프, 차트, 이미지, 배치 및 화면을 표시한다.<br>
> - **API 연동** : 백엔드에서 받은 **데이터를 화면에 시각화** 하여 표현한다.

## 프론트엔드와 혼용되는 용어

> - **클라이언트 사이드(Client-side)** : 사용자의 기기(브라우저)에서 실행된다는 의미에서 같은 영역을 지칭한다.<br>
> - **사용자 인터페이스 (UI) / 사용자 경험 (UX)** : 프론트엔드가 구현하는 **핵심 대상** <br>
> - **웹 퍼블리싱 (Web Publishing)** : 프론트엔드 업무 중 HMTL/CSS를 이용해 **마크업 하는 영억**을 뜻한다.

<hr>

## 프론트 엔드와 백엔드의 차이점

### 프론트엔드

> - **담당 영역** : 브라우저(Client)에서 **보이는 화면 (HTML,CSS,JS,React 등)** 이 있다. <br>
> - **핵심 목표** : 편리한 사용자 경험 (UX)과 디자인을 구현한다.<br>
> - **주요 작업** : 웹 / 앱의 레이아웃, 버튼, 폼, 애니메이션, API 연동 등이 있다. <br>
> - **기술 스택** : HTML5, CSS3, JavaScript, React,Vue.js,Next.js 등이 있다. <br>

### 백엔드

> - **담당 영역** : **서버 및 데이터 베이스 등을 설계하고 개발 및 운영**을 담당한다. <br>
>   **핵심 목표** : 데이터의 **논리적 처리와 시스템의 안전성 및 효율성**을 확보한다. <br>
> - **주요 작업** : 회원가입 / 로그인, DB 조회 및 저장, API 개발, 서버 배포 등이 있다. <br>
> - **기술 스택** : Java / Spring, Python / Django, Node.js/Express, PHP, SQL / NoSQL 등이 있다.

## 프론트 엔드와 디자인의 차이점

| 구분      | 프론트 엔드                              | 디자인                             |
| --------- | ---------------------------------------- | ---------------------------------- |
| 주요 역할 | 디자인 구현, 기능 구현, 인터랙션         | 화면 기획, GUI(시각적 요소) 제작   |
| 핵심 도구 | HTML, CSS, JavaScript, React/Vue/Angular | Figma, Sketch, Photoshop, Adobe XD |
| 집중 분야 | 기능성, 성능, 웹 접근성, 브라우저 호환성 | 사용자 경험(UX), 시각적 미학(UI)   |
| 작업 형태 | 코드 작성, 실제 동작하는 웹페이지 빌드   | 프로토타입, 와이어프레임 생성      |

## 헙업 구조

### 프론트 엔드 & 백엔드

<hr>

### 핵심 단계

> - **기획 / 설계 (초기)** : 어떤 데이터가 필요한지 (요청, 응답 구조)를 정의한다. 이 단게에서 **API 명세서(API Contract)** 를 작성하여 서로의 작업 기준을 세운다. <br>
> - **개발 (진행)** : 백엔드는 RESTful API나 GraphQL을 통해 데이터를 저공하고, 프론트엔드는 이 **API를 연결**하여 화면을 구성한다. <br>
> - **연동 / 테스트** : 실제 환경에서 프론트가 백엔드 서버를 통해 데이터를 조회 / 수정 / 삭제하는 기능을 확인한다.

<hr>

### 주요 협업 및 기술 구조

> - **API 명세 (API Contract)** : API URL 구조, 데이터 형태 (JSON) 등을 미리 정의하여 **병렬 작업**을 유도한다. <br>
> - **BFF (Backend For Frontend) 패턴** : 프론트엔드의 **요구에 특화된 백엔드 레이어**를 구성하여 효율적인 데이터 처리 및 성능을 최적화 한다. <br>
> - **JSON 기반 데이터 교환** : 프론트 엔드와 백엔드 간의 데이터 교환을 위한 **표준 형식**이다. <br>
> - **버전 관리 (Git)** : 코드를 공유하고 통합하는 도구이다.

<hr>

### 성공적인 협업 팁

> - **소통 단절 방지** : 백엔드 개발자도 프론트엔드 작업의 UI 요소를 이해하고, 프론트엔드도 데이터 구조를 이해해야한다. <br>
> - **빠른 연동 테스트** : 개발 초기에 **Mock API(가짜 데이터)**를 활용하여 백엔드가 왼료되기 전에도 프론트가 개발을 진행 할 수 있도록 한다.

<hr>

### 프론트엔드 & 디자인

> - **시스템 기반 협업** : 디자인 토큰(색상, 간격, 폰트)과 재사용 가능한 컴포넌트를 정의하여 **디자이너의 의도와 개발 구현간의 불일치를 최소화**한다. <br>
> - **도구 및 언어 공유** : _[Figma]_ 를 주 소통 도구로 사용하며, 디자인 언어를 코드 (CSS/JS)로 바로 변환할 수 있는 환경을 구축한다. <br>
> - **기술적 설계 협업** : FSD(Feature Sliced Design) 같은 기능 중심의 아키텍처를 도입하여 **도메인 기반**으로 폴더 구조를 나눈고 독립성을 확보한다. <br>
> - **개발 과정의 커뮤니케이션** : 개발자는 디자인 레퍼런스(Codepen 등)를 요구하거나 컴포넌트 단위로 구현하여 중간 체크를 통해 **수정 비용을 절감**한다. <br>
> - **프로세스** : 기획 / 설계(디자인 시스템 적용) -> 디자인 -> FE 프로토타입 / 개발 -> 통합 테스트 및 QA 순으로 진행된다.
